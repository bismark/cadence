Issue: PlayerScreen.handlePlayPause always calls seekToSpan() (clipBeginMs+1) when resuming, which jumps back to span start even if user paused mid-span.

Fix: When positionMs is already within a timed span, just resume playback without seeking. Only seek when outside any timed span.

Reference: player/app/src/main/java/com/cadence/player/ui/PlayerScreen.kt handlePlayPause.
