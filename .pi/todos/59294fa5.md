## Issue
Audio files are extracted and probed sequentially via ffprobe. For audiobooks with 50+ chapters, this adds compile-time latency.

## Current Code
```typescript
// Sequential processing
for (const audioFile of audioFiles) {
    // extract file
    // probe duration with ffprobe
}
```

## Fix
Parallelize with `Promise.all`:
```typescript
const probeResults = await Promise.all(
    audioFiles.map(async (audioFile) => {
        // extract and probe in parallel
    })
);
```
